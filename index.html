NotebookLM to PPTX [雙模式版]

將 NotebookLM 匯出的 PDF 轉換為可編輯的 PowerPoint 簡報

v2.3-dual-mode OCR 雙模式：Lite（快速省額度）/ 標準（完整樣式）

⚡ 雙模式版本：OCR 模型可選擇（Lite 快速省額度 / 標準完整樣式）

在選擇頁面時可切換 OCR 模型，根據需求平衡速度與品質 - 點擊查看詳細對比

版本特色：此版本使用 gemini-2.5-flash-lite 模型進行 OCR 文字提取，在保持內容準確的同時大幅節省免費 API 額度。

模型配置（並行處理）：
• 文字移除：gemini-2.5-flash-image (標準版，暫無 lite 版本)
• OCR 提取：gemini-2.5-flash-lite (輕量版，預設)
⚡ 兩者同時執行，節省處理時間

✅ 優勢：
• 並行處理 → 速度提升約 40-50%
• OCR 使用 lite 模型 → 節省約 50% API 額度
• 文字內容和位置識別仍然準確
⚠️ 不適合需要保留視覺層次的簡報（標題大小、粗體強調、顏色區分等）

⚠️ 確認限制：Lite 模型完全無法偵測字體樣式（字型大小、粗體、顏色全部缺失）。匯出的 PPTX 所有文字將使用統一樣式，失去原始設計的視覺層次。
✅ 保留：文字內容、位置、對齊方式 | ❌ 缺失：字體大小、粗細、顏色、行高

📋 使用建議：
• 適合場景：純文字筆記、會議記錄、內容草稿（不在意視覺格式）
• 不適合：精美簡報、品牌展示、教學投影片（需要視覺層次）
• 若需完整樣式：請使用標準版 index.html

避開 gen-lang-client Bug

如果你的 API Key 專案名稱包含 gen-lang-client，可能受到已知 Bug 影響導致無法使用。
解決方法：在 Google Cloud Console 建立自訂名稱的新專案（如 "my-notebook-pptx"），然後在該專案下建立 API Key。

如何取得 API Key：

  1. 前往 Google Cloud Console 建立新專案（自訂名稱）
  2. 前往 Google AI Studio
  3. 點擊「Create API Key」並選擇你剛建立的專案
  4. 複製產生的 API Key
